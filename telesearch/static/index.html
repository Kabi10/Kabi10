Setup Guide & How-To
What This Tool Does

TG Search lets you discover, save, and organize public Telegram channels and groups in a personal index. Unlike other crawlers, it never joins groups and never creates bots. It uses a Telethon MTProto session to perform safe, read-only global searches.

Architecture: Telethon MTProto client → messages.SearchGlobalRequest (paginated global search with 10x more results) → get_entity (username resolve) → SQLite tg_index + flagged_groups tables. No messages fetched. No membership changes.
Setup Checklist
  • TG Search running on http://127.0.0.1:8001
  • Telegram session file session exists (for MTProto auth)
  • SQLite database telesearch.db with tg_index table initialized
  • Telegram client connected (the Search tab will show an error if not)
First run: Start with python main.py. The lifespan manager auto-connects the Telethon client. Open http://127.0.0.1:8001 and try a keyword search. If you get "Telegram client not connected", verify your session file exists and is valid.
Using the Search Tab
  • Type a keyword and press Enter or wait 600ms. Searches use Telegram's native global search.
  • Filter by type: All, Channels, Groups, or Bots.
  • Sort: Relevance (Telegram's ranking), Members (descending), or Smallest First (ascending — great for finding small/private groups).
  • Member range: Set Min/Max to filter by subscriber count.
  • Save to index: Click the Save button on any card. It stores the group metadata to your personal SQLite index.
No ban risk: We use messages.SearchGlobalRequest with pagination and 3-second delays between pages, plus a background scanner with 5-second delays. These are legitimate Telegram API methods — no scraping, no automation abuse. You are not joining or messaging anything.
Using My Index
  • Filter & search: Use the category dropdown, type filter, status filter, and search box to quickly find saved items.
  • Edit items: Click Edit to set a custom category, add tags (comma-separated, clickable to remove), and write private notes.
  • Delete items: Click Delete to remove from your index. This does not affect the actual Telegram group.
  • Re-check All: Click the button to verify every saved username is still resolvable. Dead links get marked INACTIVE with a red badge and dimmed card. Active ones show ACTIVE in green.
Dead-link check: For each saved item, we call client.get_entity(username) with a 0.6s delay. If Telegram returns an error (banned, deleted, private), we mark it inactive. This is a read-only operation — we do not join, leave, or interact.
Data Storage

Everything is stored locally in your telesearch.db SQLite file. The tg_index table contains:

  • telegram_id, title, username, type, members — fetched from Telegram search
  • category, tags, notes — your personal organization
  • is_active, last_checked — dead-link tracking
Keyboard Shortcuts
  • Enter in search box — trigger search immediately
  • Esc — close any open modal (edit, guide)
